## Windows node: HR Excel → PDF MVP
Goal: allow Telegram-driven HR-style repetitive work (convert Excel files to PDF) on a Windows PC using an OpenClaw node.

### Key learnings
- **Control UI approvals are not always available for Windows nodes.**
  - If the node does not advertise `system.execApprovals.get/set`, the Control UI cannot show pending approvals.
  - In that scenario, any command that requires a prompt results in `approval timed out`.
- **Inline shell wrappers are brittle under allowlist mode** (examples: `powershell -Command ...`, `cmd /c ...`).
  - These commonly show up as `allowlist miss`.
- **Best practice:** allowlist a fixed executable/script and run with `-File`.

### Working approach (MVP)
1) Configure node host exec approvals on Windows via:
   - `C:\Users\<user>\.openclaw\exec-approvals.json`
   - set `security=allowlist`, `ask=off`, `askFallback=allowlist` for agent `main`
   - allowlist `powershell.exe` (full path)
   - allowlist a fixed script: `C:\Users\<user>\Desktop\xlsx-to-pdf.ps1`
2) Use a script that converts files from:
   - `Desktop\HR\IN` → `Desktop\HR\OUT` and moves processed XLSX to `Desktop\HR\DONE`
3) Execute via node:
   - `powershell.exe -NoProfile -ExecutionPolicy Bypass -File C:\Users\<user>\Desktop\xlsx-to-pdf.ps1`

### Automation scheduling
- Added a 30-minute automation tick (Gateway cron) intended to run the HR converter periodically:

## Ops UX: dashboard + watchdog
To reduce "silent failures" (node offline, tunnel closed, approvals blocking), added:
- **Telegram Dashboard** message updated by cron every 30 minutes (`telegram-dashboard-tick`).
- **Node watchdog** cron to alert on node offline/online transitions (`node-watchdog`).

## Security notes
- Do **not** paste or commit any tokens (gateway token, approvals socket tokens, dashboard URLs containing `#token=`).
- Prefer narrow allowlists (script/exe) over allowing full shells.
